## 🧩 第二部分：前端基础知识（深水区）

### 1. JavaScript 引擎级理解

#### 1.1 执行上下文与作用域

- **全局上下文、函数上下文、eval 上下文**
- **变量对象 (VO) 与活动对象 (AO)**：ES3 规范，现在用词改为 Environment Record
- **作用域链**：[[Scopes]] 属性，闭包的底层实现（函数定义时保存作用域链）
- **闭包**：
  - 形成条件：函数嵌套 + 内部函数引用外部变量
  - 内存泄漏场景：被闭包引用的 DOM 元素无法回收
  - 解决方案：将闭包引用置为 `null`，或使用 WeakMap 存储
- **this 绑定**：
  - 默认绑定（非严格模式全局，严格模式 undefined）
  - 隐式绑定（对象调用）
  - 显式绑定（call/apply/bind）
  - new 绑定（优先级最高）
  - 箭头函数（词法作用域，无 this 绑定）
  - 手写：`call`、`apply`、`bind`（需处理边界）

#### 1.2 事件循环（Event Loop）

- **浏览器模型**：
  - 宏任务：script 整体、setTimeout、setInterval、setImmediate（非标准）、I/O、UI 渲染、postMessage、MessageChannel
  - 微任务：Promise.then、MutationObserver、queueMicrotask、process.nextTick（Node）
  - 执行顺序：同步代码 -> 微任务（清空整个微任务队列）-> 宏任务（取一个执行）-> 渲染（requestAnimationFrame 在渲染前）
  - 微任务中产生新微任务会继续在当前宏任务末尾执行
- **Node 模型**（libuv）：
  - 六个阶段：timers -> pending callbacks -> idle, prepare -> poll -> check -> close
  - `process.nextTick` 不属于任何阶段，在当前阶段结束后立即执行
  - `setImmediate` 属于 check 阶段
  - 浏览器与 Node 差异（面试高频）
- **requestAnimationFrame**：在浏览器每次重绘前执行，适合动画
- **requestIdleCallback**：在空闲时间执行，优先级低

#### 1.3 Promise 与异步编程

- **Promise/A+ 规范**：then 链式调用、异步执行、状态不可逆
- **手写完整 Promise**：需实现 then、catch、finally、resolve、reject、all、race、allSettled、any
  - 注意点：then 的异步性（微任务模拟）、onFulfilled/onRejected 的异常捕获
- **async/await**：本质是 Generator + Promise 的语法糖，co 模块原理
- **事件订阅模式**：手写 EventEmitter（on、off、emit、once）

#### 1.4 ES6+ 核心特性

- **模块化**：
  - ES Module vs CommonJS 差异（编译时静态分析、动态导入、循环引用处理）
  - `import()` 动态加载原理
- **Proxy 与 Reflect**：
  - Proxy 能拦截的操作（13 种）
  - 实现数据劫持（对比 Object.defineProperty 的缺陷：无法监听新增属性、数组索引）
- **Symbol**：应用场景（唯一属性、迭代器、Symbol.hasInstance 等）
- **Set/Map/WeakSet/WeakMap**：弱引用特点、内存泄露避免

### 2. 浏览器与渲染原理

#### 2.1 浏览器架构

- **进程与线程**：浏览器主进程、渲染进程（Renderer）、GPU 进程、网络进程、插件进程
- **站点隔离（Site Isolation）**：Chrome 安全特性
- **渲染进程**：主线程（JS 执行、解析 DOM/CSS、布局、绘制）、合成线程、光栅化线程、Worker 线程

#### 2.2 导航流程（从 URL 到页面显示）

1. **输入处理**：搜索 vs URL
2. **DNS 解析**：缓存 -> 递归/迭代查询
3. **TCP 连接**：三次握手（涉及 HTTP/1.1 keep-alive、HTTP/2 多路复用）
4. **TLS 握手**（HTTPS）：非对称加密协商密钥 -> 对称加密传输
5. **请求与响应**：状态码、缓存判断（强缓存、协商缓存）
6. **解析 HTML**：预解析扫描器 -> 构建 DOM 树
7. **加载 CSS**：不会阻塞 DOM 解析，但阻塞渲染（CSSOM 构建完成才合成渲染树）
8. **加载 JS**：默认阻塞解析（defer/async 区别）
9. **渲染过程**：
   - 构建 DOM 树
   - 构建 CSSOM 树
   - 合并为渲染树（Render Tree，只包含可见节点）
   - 布局（Layout）：计算几何信息
   - 分层（Layer）：提升为合成层（will-change、transform、video、canvas 等）
   - 绘制（Paint）：记录绘制指令
   - 合成（Composite）：将图层分块、光栅化、合成为位图，显示
10. **加载完成事件**：DOMContentLoaded（DOM 树构建完成）、load（所有资源加载完成）

#### 2.3 重绘与回流

- **触发条件**：
  - 回流：几何属性变化、添加/删除元素、窗口大小变化、获取某些属性（offsetTop, scrollTop 等）
  - 重绘：颜色、可见性等不影响布局的属性
- **优化策略**：
  - 批量修改样式（class、cssText）
  - 读写分离（将读操作提前缓存）
  - 脱离文档流（绝对定位、fixed）
  - 利用合成层（transform、opacity 不触发回流）
  - 使用 `requestAnimationFrame` 读写

#### 2.4 事件系统

- **事件流**：捕获阶段（从 window 到目标）-> 目标阶段 -> 冒泡阶段
- **事件委托**：利用冒泡，减少监听器；注意 focus/blur 无冒泡，可捕获
- **合成事件（React）**：为什么设计？抹平浏览器差异、提升性能（事件池）、React17 挂载到 root 而非 document
- **自定义事件**：`new CustomEvent()`、`dispatchEvent`

### 3. 网络与安全

#### 3.1 HTTP 协议演进

- **HTTP/1.0**：短连接、无状态
- **HTTP/1.1**：
  - 持久连接（Connection: keep-alive）
  - 管道化（但队头阻塞未解决）
  - 增加 Host 头、支持断点续传
- **HTTP/2**：
  - 二进制分帧（帧、流）
  - 多路复用（解决队头阻塞，但仍存在 TCP 层队头阻塞）
  - 头部压缩（HPACK）
  - 服务器推送
- **HTTP/3**：基于 QUIC（UDP），彻底解决队头阻塞、0-RTT

#### 3.2 HTTPS

- **加密过程**：TLS 握手（1.2 和 1.3 差异）
  - 客户端发送支持的加密套件、随机数
  - 服务器返回证书、公钥、随机数
  - 客户端验证证书，生成预主密钥，用公钥加密发送
  - 双方计算会话密钥
  - 切换加密通信
- **中间人攻击**：如何防范（证书链验证、公钥固定 HPKP）

#### 3.3 TCP 基础

- **三次握手**：SYN, SYN-ACK, ACK；为什么不是两次？（防止旧连接请求）
- **四次挥手**：FIN, ACK, FIN, ACK；为什么 TIME_WAIT 持续 2MSL？（保证最后 ACK 到达、防止旧数据包干扰）
- **滑动窗口**：流量控制机制
- **拥塞控制**：慢启动、拥塞避免、快重传、快恢复

#### 3.4 前端安全

- **XSS**（跨站脚本）：
  - 反射型、存储型、DOM 型
  - 防御：输入过滤、输出编码（HTML 实体、JS 编码）、CSP（Content-Security-Policy）、HttpOnly Cookie
- **CSRF**（跨站请求伪造）：
  - 防御：SameSite Cookie（Lax/Strict）、CSRF Token、Referer 校验、二次验证
- **点击劫持**：X-Frame-Options: DENY/SAMEORIGIN
- **中间人攻击**：强制 HTTPS、HSTS

## ⚛️ 第三部分：React 生态（必考核心）

### 1. React 理念与架构

- **声明式 UI**：UI = render(state)
- **组件化**：组合优于继承
- **Virtual DOM**：
  - 为什么要有？跨平台、最小化操作真实 DOM（批量更新）
  - Diff 算法 O(n) 策略：同层比较、不同节点类型直接替换、key 优化

### 2. Fiber 架构（P7 必问）

- **背景**：旧版 Stack Reconciler 递归不可中断，掉帧
- **Fiber 节点**：每个组件对应一个 fiber，保存类型、状态、副作用、链表结构
- **双缓存技术**：workInProgress tree 和 current tree
- **可中断渲染**：时间切片（time slicing），利用 `requestIdleCallback` 模拟
- **优先级调度**：Lane 模型
- **渲染阶段（Render Phase）**：构建 workInProgress 树，可中断
- **提交阶段（Commit Phase）**：一次性同步执行 DOM 操作，不可中断
- **副作用（Effect）**：`useEffect` 与 `useLayoutEffect` 执行时机

### 3. Diff 算法细节

- **节点比较**：
  - 类型不同：直接替换
  - 类型相同：更新属性
  - 子节点比较：双端对比（新旧链表首尾）
- **key 的作用**：识别节点移动、复用；不要用数组索引作为 key（会破坏稳定性）
- **函数组件 vs 类组件 diff**：本质相同

### 4. Hooks 原理

- **Hooks 链表**：每个 Fiber 节点有 `memoizedState` 指向 hook 链表
- **`useState`**：调用 `setState` 触发更新，dispatchAction 加入更新队列
- **`useEffect`**：异步执行，在布局绘制后执行；`useLayoutEffect` 同步执行（类似 componentDidMount）
- **`useMemo` / `useCallback`**：依赖项比较，如果不变则返回上次缓存值
- **为什么不能在条件语句中使用 Hooks**：保证每次渲染顺序一致，链表访问正确
- **自定义 Hooks**：复用状态逻辑

### 5. 状态管理

- **Redux**：
  - 单一数据源、state 只读、纯函数修改
  - 中间件原理：`applyMiddleware` 组合 `dispatch`
  - `redux-thunk`：函数形式的 action
  - `redux-saga`：ES6 Generator，监听模式
  - Redux Toolkit：简化样板代码
- **MobX**：响应式代理，自动收集依赖
- **Context + useReducer**：适合中小项目，但需注意穿透渲染优化（`React.memo` + `useMemo`）
- **Zustand / Jotai**：轻量、原子化

### 6. 性能优化

- **减少渲染**：
  - `React.memo`（浅比较 props）
  - `PureComponent`（类组件）
  - `useMemo` 缓存值，`useCallback` 缓存函数
- **减少计算**：`useMemo` 避免昂贵计算
- **代码分割**：`React.lazy` + `Suspense`，配合 Webpack 动态 import
- **SSR/SSG**：Next.js 数据获取（`getServerSideProps` / `getStaticProps`）
- **长列表优化**：`react-window` / `react-virtualized`

## 🏗️ 第四部分：工程化与构建

### 1. Webpack 核心原理

- **构建流程**：初始化 -> 编译（make）-> 构建模块（build-module）-> 封装（seal）-> 输出（emit）
- **Loader**：
  - 执行顺序：从右到左，从下到上
  - 开发 Loader：返回字符串/JS 代码
- **Plugin**：
  - 监听 Webpack 生命周期（tapable）
  - 开发 Plugin：`apply(compiler)` 注册 hooks
- **热更新（HMR）**：
  - 建立 WebSocket 连接，监听文件变化
  - 通知客户端更新，通过 `module.hot.accept` 执行回调
- **Tree Shaking**：
  - 依赖 ES6 模块静态结构
  - `sideEffects` 配置避免误删
- **代码分割**：
  - 入口起点、动态 import、SplitChunksPlugin

### 2. Vite 原理

- **开发环境**：基于 esbuild 预构建依赖，利用浏览器原生 ES Module
- **生产环境**：Rollup 打包
- **优势**：启动快、HMR 快
- **对比 Webpack**：开发无打包，按需编译

### 3. Babel

- **编译流程**：解析（parse）-> 转换（transform）-> 生成（generate）
- **插件与预设**：preset-env（根据目标环境转换语法）
- **AST 操作**：常见节点类型

### 4. 性能监控与指标

- **核心 Web Vitals**：
  - LCP（最大内容绘制）：2.5s 以内
  - FID（首次输入延迟）：100ms 以内
  - CLS（累计布局偏移）：0.1 以内
- **其他指标**：FP、FCP、TTI、TBT
- **监控工具**：Lighthouse、Performance API、Web Vitals 库
