# 题目名
Vue `computed` 与 `watch` 的区别

## 题目描述
`computed` 与 `watch` 分别适合解决什么问题？它们的执行时机、缓存行为有什么差异？如何选择？

## 题目答案
`computed`：
- 用于“由已有状态派生出新状态”，本质是带缓存的 getter。
- 只有依赖变更时才会重新计算；多次读取不会重复执行。
- 更适合表达“数据之间的映射关系”。

`watch`：
- 用于“观察某个状态变化并执行副作用”。
- 适合异步操作、请求、手动操作 DOM、写日志/埋点等。
- 默认不会缓存结果（它的目的不是产出值，而是触发回调）。

选择建议：
- 需要一个可在模板/逻辑中重复使用的派生值：用 `computed`。
- 需要在变化时执行副作用：用 `watch`。
