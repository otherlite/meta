# 题目名
React 受控与非受控组件

## 题目描述
什么是受控组件与非受控组件？各自优缺点是什么？在表单场景下如何选择？如何处理性能问题（大量输入、频繁 setState）？

## 题目答案
受控组件：
- 表单值由 React state 驱动，输入变化触发 `onChange` 更新 state。
- 优点：数据源唯一，易做校验、联动与回放。
- 缺点：频繁更新可能带来渲染压力（尤其是大表单）。

非受控组件：
- 表单值由 DOM 自己维护，React 通过 `ref` 在需要时读取。
- 优点：输入过程更少触发渲染，性能更好。
- 缺点：状态分散，联动与校验需要额外处理。

选择建议：
- 强联动、强校验、需要随时拿到最新值：偏受控。
- 超大表单、输入性能敏感、只在提交时读取：可用非受控或混合方案。

性能治理：
- 局部状态下沉（把 state 放到最小影响范围）。
- 批量更新与节流校验。
- 对复杂表单用专用表单库并理解其订阅模型。
