# 题目名
Vue2/Vue3 响应式原理与 diff

## 题目描述
Vue2 与 Vue3 的响应式实现分别是什么？各自的优缺点是什么？Vue 的 diff/patch 过程大致做了什么优化？

## 题目答案
响应式原理：
- Vue2：基于 `Object.defineProperty` 劫持 getter/setter。
  - 优点：实现直观，生态成熟。
  - 局限：
    - 无法天然监听新增/删除属性（需要 `Vue.set`/`Vue.delete`）
    - 无法直接监听数组索引与长度变化（通过重写数组方法做补丁）
    - 深层对象需要递归劫持，初始化成本高
- Vue3：基于 `Proxy`。
  - 优点：
    - 能监听新增/删除属性、数组索引/长度等
    - 代理是懒执行的（访问到才代理/追踪），整体更灵活
  - 局限：旧环境兼容性要求更高（需要 Proxy 支持）。

依赖收集与触发：
- 读取时 track（把当前 effect 收集到依赖集合）
- 写入时 trigger（通知相关 effect 重新执行）

diff/patch（概念）：
- 对比新旧虚拟节点树，找出最小变更集合并更新到真实 DOM。
- 关键优化点：
  - 同层比较与 key：列表更新依赖稳定 key，减少不必要的移动/重建。
  - 静态提升（Vue3）：把静态节点提升为常量，减少每次渲染创建。
  - Patch Flag：编译阶段标记动态部分，运行时跳过无关 diff。
