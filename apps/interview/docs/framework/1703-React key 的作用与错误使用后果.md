# 题目名
React `key` 的作用与错误使用后果

## 题目描述
React 列表渲染里 `key` 的作用是什么？为什么不推荐用数组下标做 key？会导致哪些具体问题？

## 题目答案
`key` 用于帮助 React 在同层级的子元素之间建立稳定身份，从而在 diff 时复用正确的节点，减少不必要的卸载/重建。

为什么不推荐用下标当 key：
- 当列表发生插入/删除/排序时，下标会变化，React 可能把“旧节点状态”错误地复用到“新位置的不同数据”上。

可能的后果：
- 输入框内容错位：用户在第 N 行输入，插入一行后输入跑到别的项。
- 动画/展开状态错乱：某项的局部 state（展开、选中）被错误复用。
- 性能问题：错误的 key 让 React 频繁卸载/重建，导致更多 DOM 操作。

最佳实践：
- `key` 选择“稳定且唯一”的标识（数据库 id、业务唯一值）。
- 如果列表只做 append（只在末尾追加）且不重排，下标 key 的问题会少一些，但仍不推荐作为默认方案。
