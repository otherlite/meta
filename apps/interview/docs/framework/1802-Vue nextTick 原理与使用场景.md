# 题目名
Vue `nextTick` 原理与使用场景

## 题目描述
Vue 的 `nextTick` 是做什么的？它为什么能在 DOM 更新后拿到最新 DOM？Vue2 与 Vue3 的实现思路有什么差异？

## 题目答案
`nextTick` 用于在“数据变更触发视图更新”之后执行回调，确保能拿到更新后的 DOM。

为什么需要它：
- Vue 的 DOM 更新是异步批处理的：同一事件循环内多次数据变更会被合并成一次渲染。
- 立即读取 DOM 可能读到旧状态。

实现思路（概念）：
- 把回调放到微任务队列（优先）或宏任务队列中，在本轮更新 flush 完之后执行。
- Vue 内部会维护一个回调队列，更新完成后统一执行。

使用场景：
- 数据更新后需要测量 DOM（宽高、位置）。
- 依赖 DOM 的第三方库初始化/更新。
