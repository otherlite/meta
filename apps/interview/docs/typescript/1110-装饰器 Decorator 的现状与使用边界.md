# 题目名
装饰器（Decorator）的现状与使用边界

## 题目描述
TypeScript 装饰器是什么？它能做哪些事（类/方法/属性参数装饰）？当前标准与 TS 实现有什么差异？在工程中使用装饰器要注意什么？

## 题目答案
装饰器本质：在定义阶段对类或其成员进行“元编程增强”，例如注入元数据、拦截方法、做依赖注入等。

能做的事（概念）：
- 类装饰器：修改/替换构造函数，注册到容器。
- 方法装饰器：包装方法实现（埋点、权限、缓存）。
- 属性/参数装饰器：配合反射元数据做校验/注入。

现状与差异：
- 装饰器标准在演进，TS 历史实现与新提案在语义上有差异。
- 依赖装饰器的框架通常对编译配置有要求（如 emitDecoratorMetadata）。

工程注意：
- 会增加编译与运行时复杂度，影响 tree-shaking。
- 要评估长期维护成本与标准变动风险。
- 在核心业务上谨慎引入“隐式行为”，避免调试困难。
