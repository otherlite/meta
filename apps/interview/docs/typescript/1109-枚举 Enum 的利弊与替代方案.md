# 题目名
枚举（Enum）的利弊与替代方案

## 题目描述
TypeScript 的 `enum` 有哪些优点与坑点（运行时代码、反向映射、编译产物）？在工程里有哪些更推荐的替代方案？

## 题目答案
优点：
- 语义清晰，集中管理常量集合。
- 数字枚举有反向映射（可从值反查键）。

坑点：
- `enum` 会生成运行时代码（影响包体与 tree-shaking）。
- 数字枚举的反向映射可能带来意外行为。
- 与跨包/多端协议同步时容易出现版本不一致。

替代方案：
- `as const` + 字面量联合：
  - 用对象/数组定义常量，并导出 `type` 为联合类型。
- 对外协议：用字符串字面量更稳定（可读性更好、调试更友好）。

工程建议：
- 如果需要纯类型层的枚举语义，优先 `as const` + 联合；只有在确实需要运行时代码/反向映射时再用 enum。
