# 题目名
`any` / `unknown` / `never` 的区别与最佳实践

## 题目描述
请说明 TypeScript 中 `any`、`unknown`、`never` 的区别；在工程中分别适用于什么场景？

## 题目答案
`any`：
- 关闭类型检查的“逃生舱”。对 `any` 做任何操作都不会报错。
- 风险：会污染类型系统（`any` 参与推导会让下游也变成 `any`）。
- 场景：临时迁移、第三方库缺类型且短期无法补齐（但应尽量局部化）。

`unknown`：
- “安全的 any”。你可以把任何值赋给 `unknown`，但不能直接把 `unknown` 当成具体类型使用，必须先做类型收窄（`typeof`、`in`、用户自定义 type guard 等）。
- 场景：
  - 处理外部输入（`JSON.parse`、接口返回、用户输入）
  - 作为泛型默认上界，强迫使用者显式处理类型

`never`：
- 表示“不可能发生”的类型（空集合）。
- 常见来源：
  - 函数永不返回（抛错/死循环）
  - 联合类型被穷尽后剩余分支
- 场景：
  - `switch` 穷尽检查（exhaustiveness check），确保新增枚举值时编译报错

工程建议：
- 能用 `unknown` 就别用 `any`。
- 用 `never` 做分支穷尽，提升重构安全性。
