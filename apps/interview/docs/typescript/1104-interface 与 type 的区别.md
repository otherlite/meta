# 题目名
`interface` 与 `type` 的区别

## 题目描述
TypeScript 中 `interface` 与 `type` 有哪些差异？在工程里如何选择？请结合“声明合并”“扩展方式”“表达能力”说明。

## 题目答案
二者都能描述对象结构，但定位略有差异。

主要差异：
- 声明合并：
  - `interface` 支持同名声明自动合并，适合做“可扩展的公共协议”（例如给第三方类型打补丁）。
  - `type` 不支持同名合并。
- 扩展方式：
  - `interface` 用 `extends` 扩展。
  - `type` 用交叉类型 `&` 组合。
- 表达能力：
  - `type` 能表达更复杂的类型运算：联合、条件类型、映射类型等。
  - `interface` 更偏向“对象/类的形状”。

工程选择建议：
- 需要对外暴露可扩展协议（如公共库的配置对象、全局声明扩展）时优先 `interface`。
- 需要类型计算（联合/条件/映射）或组合多个类型时优先 `type`。
- 团队统一风格也很重要：保持一致能降低沟通成本。
