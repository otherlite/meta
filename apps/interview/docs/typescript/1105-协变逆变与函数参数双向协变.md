# 题目名
协变/逆变与函数参数双向协变

## 题目描述
请解释协变（covariance）与逆变（contravariance）的概念。在 TypeScript 中函数参数为什么有“双向协变（bivariance）”的现象？这会带来什么风险，如何降低？

## 题目答案
协变/逆变用来描述“类型在包含关系下如何安全替换”。

直观理解：
- 协变：子类型可以替换父类型（保持同方向）。例如返回值类型通常是协变的：返回更具体的类型是安全的。
- 逆变：父类型可以替换子类型（反方向）。函数参数位置通常是逆变的：接受更宽泛的参数更安全。

TypeScript 里的现象：
- 在某些场景（尤其是方法参数、回调）TS 为了兼容性与可用性，对函数参数采用了更宽松的检查策略，表现为“近似双向协变”。

风险：
- 可能允许把“只接受更窄类型参数的函数”当成“接受更宽类型参数的函数”使用，运行时可能出现不符合预期的参数。

降低风险的做法：
- 开启 `strictFunctionTypes`（在 `strict` 下默认开启），它会对函数类型的参数检查更严格（注意类方法等仍可能有差异）。
- 尽量用函数类型属性而非方法签名表达回调（更容易触发严格检查）。
- 对外部回调输入做运行时校验（尤其是边界层）。
