# 题目名
结构化类型系统与“鸭子类型”

## 题目描述
TypeScript 为什么被称为结构化类型系统？它与名义类型系统有什么区别？结构化类型会带来哪些便利与风险？

## 题目答案
结构化类型系统：判断两个类型是否兼容主要看“结构是否匹配”（属性/方法形状），而不是看它们的名字或声明来源。

名义类型系统：类型兼容依赖显式的类型名/继承关系（例如某些语言里两个结构相同的类也不一定互相兼容）。

便利：
- 更贴近 JS 生态，和对象字面量/接口形状天然契合。
- 复用能力强：只要结构满足就能直接赋值/传参。

风险：
- 可能出现“看起来兼容但语义不同”的误用（结构相同但含义不同）。
- 对关键领域模型可用品牌类型（branding）或更严格的封装降低误用。
