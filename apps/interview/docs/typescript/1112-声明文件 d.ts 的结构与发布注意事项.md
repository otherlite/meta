# 题目名
声明文件（`.d.ts`）的结构与发布注意事项

## 题目描述
`.d.ts` 的作用是什么？`declare` 关键字常见用法有哪些？发布 npm 包时如何确保类型被正确消费（`types`、`exports`、`typesVersions` 等）？

## 题目答案
`.d.ts` 的作用：
- 为 JS 代码或编译产物提供类型信息，让 TS 项目获得类型检查与智能提示。

常见 `declare` 用法：
- `declare module 'xxx'`：模块声明/补类型。
- `declare global { ... }`：扩展全局类型。
- `declare namespace`：命名空间（多用于历史/全局脚本）。
- `declare const/function/class`：声明外部存在的值/函数/类。

发布注意：
- `package.json` 中声明 `types` 指向入口 `.d.ts`。
- 若使用 `exports`，需要在 `exports` 中同时声明类型路径（避免只导出 JS 入口导致类型丢失）。
- 多版本 TS 兼容：必要时用 `typesVersions`。
- 保证类型与运行时 API 一致，避免“类型能用但运行时报错”。
