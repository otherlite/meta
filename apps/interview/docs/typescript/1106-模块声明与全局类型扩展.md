# 题目名
模块声明与全局类型扩展

## 题目描述
如何在 TypeScript 中给没有类型定义的三方库补类型？如何扩展全局类型（如 `Window`）或给已有模块做声明合并？工程上有哪些注意点？

## 题目答案
常见做法：

1) 给无类型库补声明
- 使用 `declare module 'xxx'` 声明模块的导出（从最小可用开始逐步完善）。
- 如果只想先让编译通过，可以先声明粗粒度类型，但应尽快收敛到更准确的类型。

2) 扩展全局类型
- 使用 `declare global` 扩展全局命名空间，例如扩展 `Window`、`NodeJS.ProcessEnv` 等。
- 注意该声明文件必须是一个模块（通常需要 `export {}`）才能安全地使用 `declare global`。

3) 声明合并
- `interface` 同名会合并，可用于给已存在的接口追加字段（如扩展 `Window`）。
- 某些库暴露的类型也可通过 module augmentation 做扩展（`declare module 'lib' { interface Options { ... } }`）。

注意事项：
- 类型补丁应放在可被 TS 编译器包含的路径下（受 `tsconfig.json` 的 `include/typeRoots` 影响）。
- 避免把类型扩展做得过于宽泛（如大量 `any`），会降低类型系统价值。
- 尽量贴近真实运行时行为，避免“类型正确但运行时崩”的错觉。
