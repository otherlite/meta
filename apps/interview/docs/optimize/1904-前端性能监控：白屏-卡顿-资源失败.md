# 题目名
前端性能监控：白屏、卡顿与资源失败

## 题目描述
线上如何监控白屏、卡顿、资源加载失败？分别有哪些可行的检测/上报思路？如何避免误报？

## 题目答案
白屏监控：
- 思路：在首屏关键时刻检查页面是否有有效内容（例如关键容器是否有渲染、首屏节点数量、首屏截图比对等）。
- 常用：结合性能时序（FCP/LCP）与路由信息，上报“长时间无渲染”的异常。

卡顿监控：
- 采集长任务与帧率信号：
  - 通过 Performance API/Observer 观察长任务
  - 或用 `requestAnimationFrame` 采样估算掉帧（注意成本与误差）
- 结合用户交互事件与 INP 指标定位热路径。

资源失败：
- 监听资源 `error`（捕获阶段）收集 script/link/img 加载失败。
- 上报资源 URL、类型、状态码（若可得）、CDN 节点等。

避免误报：
- 采样与阈值：只对关键路由/关键用户采样，设置合理阈值。
- 结合重试结果：资源失败可先重试再上报。
- 与版本强绑定：没有版本信息的监控很难治理。
