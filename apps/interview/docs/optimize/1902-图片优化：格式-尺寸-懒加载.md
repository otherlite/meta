# 题目名
图片优化：格式、尺寸与懒加载

## 题目描述
前端图片优化有哪些常用手段？如何选择图片格式（WebP/AVIF/JPEG/PNG/SVG）？如何设置合适尺寸与响应式图片？懒加载要注意什么？

## 题目答案
优化目标：减少下载体积、减少解码/绘制开销、提升 LCP。

格式选择（概念）：
- AVIF/WebP：压缩率高，适合照片类（注意兼容与编码成本）。
- JPEG：照片类传统方案。
- PNG：需要无损/透明的位图。
- SVG：矢量图标/简单图形。

尺寸与响应式：
- 不要把超大图缩放展示：应提供与展示尺寸接近的资源。
- 用 `srcset/sizes` 提供不同 DPR/宽度的版本，让浏览器自动选择。

懒加载：
- 原生 `loading="lazy"` 适合非首屏图片。
- 首屏关键图不要 lazy，必要时用 `preload` 或提升优先级。
- 注意占位尺寸，避免 CLS。
