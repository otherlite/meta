# 题目名
长任务（Long Task）与主线程阻塞的治理

## 题目描述
什么是长任务？为什么会影响交互指标（INP）与掉帧？如何在工程中定位长任务并治理（代码拆分、Web Worker 等）？

## 题目答案
长任务：主线程上执行时间过长的任务（常见阈值是 50ms 以上的连续阻塞）。它会导致：
- 用户输入无法及时响应（INP 变差）。
- 渲染/合成被延迟，出现掉帧。

定位：
- Performance 面板录制，查看 Main 线程的长条任务与调用栈。
- 关注 Long Tasks、Layout/Paint 频次、强制同步布局。

治理：
- 拆分任务：把大循环拆成小片段（分片执行），让出主线程。
- 延后非关键计算：用 `requestIdleCallback`（需降级）或异步调度。
- Web Worker：把纯计算/解析放到 worker（注意数据传输成本，必要时用 Transferable）。
- 减少 layout thrash：批量读写 DOM，避免强制同步布局。
