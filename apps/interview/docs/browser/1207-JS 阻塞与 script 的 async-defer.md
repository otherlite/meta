# 题目名
JS 阻塞与 `<script>` 的 `async/defer`

## 题目描述
为什么 JS 会阻塞 HTML 解析？`<script>` 的 `async` 与 `defer` 有何区别？执行顺序与适用场景分别是什么？

## 题目答案
为什么阻塞：
- 浏览器解析 HTML 时遇到普通 `<script>` 会暂停解析，先下载并执行脚本。
- 因为脚本可能会修改 DOM（例如 `document.write`、插入节点），需要保证执行时机一致。

`defer`：
- 脚本并行下载，不阻塞 HTML 解析。
- 执行时机：在 DOM 解析完成后、`DOMContentLoaded` 之前。
- 多个 defer 脚本按文档顺序执行。
- 适用：大多数业务脚本（依赖 DOM 结构）。

`async`：
- 脚本并行下载，不阻塞 HTML 解析。
- 执行时机：下载完成立刻执行，可能打断解析。
- 多个 async 脚本执行顺序不保证。
- 适用：独立的第三方脚本（埋点、广告、无需依赖 DOM 顺序）。
