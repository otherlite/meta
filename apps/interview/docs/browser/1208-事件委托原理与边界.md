# 题目名
事件委托（Event Delegation）原理与边界

## 题目描述
什么是事件委托？为什么把监听器绑在父节点上也能处理子节点事件？它的好处与缺点是什么？有哪些边界情况需要注意？

## 题目答案
原理：
- DOM 事件在大多数情况下会冒泡（从目标节点向上到 document/window）。
- 父节点监听事件后，通过 `event.target`/`event.currentTarget` 判断真实触发源并做分发。

好处：
- 减少监听器数量，降低内存与注册成本。
- 天然支持动态插入的子节点（无需重新绑定）。

缺点与边界：
- 不是所有事件都冒泡（如部分 focus 相关，需用捕获或替代事件）。
- 需要做目标匹配（选择器匹配）会有一定开销。
- 停止冒泡（`stopPropagation`）会影响委托链路。
