# 题目名
浏览器进程/线程模型与渲染进程任务

## 题目描述
请说明现代浏览器（以 Chromium 为例）的多进程架构：浏览器进程、渲染进程、网络进程、GPU 进程分别做什么？为什么需要多进程？

## 题目答案
多进程的主要目的：稳定性与安全隔离。

典型进程职责（概念回答）：
- 浏览器进程（Browser Process）：
  - 管理标签页、地址栏、前进后退、书签等 UI
  - 协调其他进程（调度、权限、安全策略）
- 渲染进程（Renderer Process，通常每个站点/标签页一或多个）：
  - 解析 HTML/CSS，执行 JS
  - 布局、绘制与合成相关工作（部分与 GPU 协作）
  - DOM、V8、事件循环主要发生在这里
- 网络进程（Network Process）：
  - 统一处理网络请求、缓存、Cookie 等
- GPU 进程（GPU Process）：
  - 图形栈与 GPU 加速合成，提升渲染性能

为什么要多进程：
- 单页崩溃不影响整个浏览器。
- 站点隔离（Site Isolation）降低恶意页面攻击面。
- 利于资源调度与并行（但也带来更高内存占用）。
