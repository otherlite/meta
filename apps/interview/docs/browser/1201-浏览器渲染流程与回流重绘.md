# 题目名
浏览器渲染流程与回流重绘

## 题目描述
从输入 URL 到页面显示发生了什么？渲染流水线中哪些步骤会触发回流（reflow/layout）与重绘（repaint）？如何优化？

## 题目答案
从 URL 到首屏（高度概括）：
- 解析 URL → DNS → TCP/TLS → 发送 HTTP 请求 → 接收响应
- 解析 HTML 构建 DOM
- 解析 CSS 构建 CSSOM
- DOM + CSSOM → Render Tree
- Layout（计算几何信息）→ Paint（绘制指令）→ Composite（分层合成）

回流（layout/reflow）：
- 影响元素几何信息（尺寸、位置、布局流）的变化会触发布局计算。
- 常见触发：修改 `width/height/padding/margin`、改变 `display`、插入/删除节点、读取强制同步布局属性（如 `offsetWidth`、`getBoundingClientRect`）导致 layout flush。

重绘（repaint）：
- 不改变几何信息但改变视觉样式（颜色、背景、阴影等）会触发重绘。

合成（composite）：
- 发生在 GPU 合成层级的变更（如 `transform`、`opacity`）通常只需合成，不需要 layout/paint（取决于是否已提升为合成层）。

优化策略：
- 用 `transform/opacity` 做动画，避免频繁 layout。
- 批量读写 DOM：先读后写，避免交替导致多次强制布局。
- 降低布局影响范围：使用 `contain`、减少复杂选择器与深层嵌套。
- 对长列表用虚拟列表；避免一次性渲染大量节点。
