# 题目名
`requestIdleCallback` 的用途与限制

## 题目描述
`requestIdleCallback` 解决什么问题？它适合做哪些任务，不适合做哪些任务？如何做降级与超时兜底？

## 题目答案
用途：
- 在浏览器空闲时执行低优先级任务，避免抢占主线程影响交互与渲染。

适合：
- 预计算、预加载、日志整理、非关键数据处理。

不适合：
- 必须立刻执行的任务（输入响应、关键渲染逻辑）。
- 对时序很敏感的任务（空闲回调触发时间不稳定）。

降级与兜底：
- 浏览器兼容不一，需要降级到 `setTimeout`。
- 使用 `timeout` 参数保证在一定时间内必须执行一次（避免长期饿死）。
