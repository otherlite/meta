# 题目名
输入 URL 到页面展示的完整链路

## 题目描述
请按顺序描述从输入 URL 到页面展示的全过程：DNS、TCP/TLS、HTTP、解析与渲染、资源加载与执行。面试时如何抓住主线并补充关键细节？

## 题目答案
主线（可按阶段回答）：

1) 网络阶段
- URL 解析 → DNS 解析（本地缓存/系统缓存/递归解析）
- 建立连接：TCP 三次握手；若 HTTPS 则进行 TLS 握手
- 发送 HTTP 请求，接收响应（可能命中 CDN/缓存）

2) 解析阶段
- 解析 HTML 构建 DOM
- 解析 CSS 构建 CSSOM
- DOM + CSSOM 生成渲染树（Render Tree）

3) 渲染阶段
- Layout：计算几何信息
- Paint：生成绘制指令
- Composite：分层合成并输出到屏幕

4) 资源与执行
- JS 下载与执行可能阻塞解析（取决于 `defer/async`）
- CSS 是渲染阻塞资源（会影响首次渲染）
- 字体/图片等会影响 LCP/CLS 等指标

答题技巧：
- 先把主线讲完整，再补“缓存、重定向、HTTP/2、多进程、渲染阻塞”等细节。
