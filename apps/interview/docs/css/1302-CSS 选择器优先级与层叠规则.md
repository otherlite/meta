# 题目名
CSS 选择器优先级与层叠规则

## 题目描述
CSS 的层叠（cascade）与优先级（specificity）如何决定最终样式？`!important`、内联样式、选择器优先级、后写覆盖的关系是什么？

## 题目答案
决定最终样式通常要综合：
1. 是否命中选择器
2. 重要性与来源（author/user-agent/user，是否 `!important`）
3. 优先级（specificity）
4. 书写顺序（同来源同优先级时后写覆盖前写）

优先级计算（常见记忆法）：
- 内联样式（style 属性）优先级高于一般选择器
- 选择器优先级按权重累加（a,b,c,d）：
  - a：是否内联（一般可理解为单独一档）
  - b：ID 选择器数量
  - c：类/属性/伪类数量
  - d：元素/伪元素数量
- 通配符 `*`、组合符（`>`、`+`、`~`）、`:where()` 不增加优先级。

`!important`：
- 在同一来源内，`!important` 会压过非 `!important`。
- 但两个 `!important` 冲突时仍按优先级与书写顺序决胜。

工程实践：
- 避免滥用 `!important`，它会破坏可维护性。
- 通过设计良好的样式层级（如 BEM、CSS Modules、约定的作用域）降低“权重战争”。
