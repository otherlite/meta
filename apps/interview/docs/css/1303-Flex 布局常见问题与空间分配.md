# 题目名
Flex 布局常见问题与空间分配

## 题目描述
请解释 Flex 的主轴/交叉轴、`flex-grow/shrink/basis` 的含义。为什么 `flex: 1` 常用？`min-width: auto` 为什么会导致子项无法缩小？

## 题目答案
Flex 基本概念：
- 主轴：由 `flex-direction` 决定（row/column）。
- 交叉轴：与主轴垂直。

空间分配三要素：
- `flex-basis`：参与分配的“基础尺寸”（默认 `auto`，通常取内容尺寸或 width/height）。
- `flex-grow`：有剩余空间时按比例放大。
- `flex-shrink`：空间不足时按比例缩小。

`flex: 1` 的常见含义：
- 等价于 `flex: 1 1 0%`（不同实现细节略有差异，但可理解为：基础为 0，允许放大/缩小，能更平均地分配剩余空间）。

为什么子项有时“死撑不缩小”：
- Flex 子项默认 `min-width: auto`，会把最小宽度设为内容的最小宽度（例如一长串不可断行文本）。
- 即使 `flex-shrink: 1`，也可能因为最小宽度限制而缩不下去。

解决：
- 给需要收缩的子项设置 `min-width: 0`（或 `overflow: hidden` + 断行策略），允许其在 flex 容器内真正缩小。
