# 题目名
响应式布局：容器查询（Container Queries）

## 题目描述
容器查询解决了媒体查询的哪些痛点？什么时候应该按“组件容器尺寸”响应而不是按视口响应？工程落地需要注意什么？

## 题目答案
媒体查询的问题：
- 只根据视口尺寸响应，组件在不同容器里复用时可控性不足。

容器查询的价值：
- 组件根据自身容器的尺寸变化调整布局，更利于组件化与可复用设计系统。

适用场景：
- 同一组件在侧栏/主栏/弹窗中需要不同布局。
- 组件需要在网格系统里自适应不同列宽。

落地注意：
- 需要为容器启用容器特性（如设置容器类型），并关注浏览器兼容。
- 规则不要过度复杂，避免难以维护。
