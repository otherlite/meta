# 题目名
移动端适配：vw / rem / 媒体查询

## 题目描述
移动端页面适配常见方案有哪些？`vw`、`rem`、媒体查询各自适用什么场景？如何处理 1px 细线、刘海屏安全区？

## 题目答案
常见方案：
- `vw`：基于视口宽度的相对单位，天然随屏幕变化。
  - 优点：无需 JS；缺点：一些复杂布局与字号控制需要额外设计。
- `rem`：基于根字体大小（`html { font-size }`）的相对单位。
  - 常见做法：用 JS 按 viewport 动态设置根字号，设计稿尺寸按 rem 换算。
  - 优点：体系化；缺点：依赖 JS，注意首屏闪动与缩放。
- 媒体查询：按断点写多套样式。
  - 优点：可精细控制；缺点：维护成本高。

1px 细线：
- 通过 `transform: scaleY(0.5)` 等方式实现（注意 DPR 与渲染差异）。
- 或使用 `border-image`、渐变等方案。

安全区（刘海屏）：
- 使用 `env(safe-area-inset-*)` 配合 padding 处理。

工程建议：
- 方案选择要结合团队基建与业务复杂度：H5 活动页常用 vw/rem；复杂业务可结合媒体查询。
