# 题目名
BFC 是什么与常见应用

## 题目描述
什么是 BFC（Block Formatting Context）？如何触发 BFC？BFC 能解决哪些经典布局问题？

## 题目答案
BFC：块级格式化上下文，是一个独立的布局区域，内部元素的布局不会影响外部，外部也不会影响内部（在特定规则下）。

触发 BFC 的常见方式（部分）：
- `overflow` 不是 `visible`（如 `auto/hidden`）
- `display: flow-root`
- `position: absolute/fixed`
- `float` 不为 `none`
- `display: inline-block`、`display: flex`、`display: grid`（它们会创建新的格式化上下文，通常也能隔离某些影响）

常见应用：
- 清除浮动：让父元素形成 BFC，从而包含浮动子元素的高度。
- 防止外边距折叠（margin collapse）：两个 BFC 之间不会发生 margin 折叠。
- 两栏布局：一侧 float，另一侧触发 BFC（如 `overflow: hidden`）可避免被浮动覆盖。

工程建议：
- 优先使用 `display: flow-root` 来创建 BFC（语义明确，副作用相对少），而不是滥用 `overflow: hidden`。
