# 题目名
响应式图片与 `object-fit` 的陷阱

## 题目描述
`object-fit: cover/contain` 分别做什么？为什么图片会被裁剪或留白？在响应式布局中如何避免图片变形、裁剪造成的关键信息丢失？

## 题目答案
`object-fit` 控制替换元素（如 img、video）内容如何适配其盒子。

- `contain`：完整显示内容，保持比例，可能留白。
- `cover`：填满盒子，保持比例，可能裁剪。

常见问题：
- 用 `cover` 时主体可能被裁掉，需要配合 `object-position` 调整裁剪焦点。
- 只设置宽高不考虑比例会导致拉伸变形。

实践建议：
- 对头像/封面这类可裁剪资源使用 `cover`。
- 对信息密集图片（海报、图表）优先 `contain` 或按比例容器。
- 结合 `srcset/sizes` 提供合适分辨率，避免下载过大图片。
