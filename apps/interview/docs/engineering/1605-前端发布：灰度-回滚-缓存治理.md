# 题目名
前端发布：灰度、回滚与缓存治理

## 题目描述
前端上线如何做灰度与快速回滚？资源缓存（CDN/浏览器）如何治理，避免“新 HTML 引旧 JS”导致白屏？

## 题目答案
灰度与回滚：
- 灰度：按用户/地域/设备/渠道分流到新版本，逐步扩大。
- 回滚：保留旧版本静态资源与入口配置，出现事故可快速切回。

缓存治理（核心是“入口与资源强一致”）：
- 静态资源用内容哈希命名（immutable），可长缓存。
- HTML 入口不做长缓存或短缓存（`Cache-Control: no-cache`），确保能快速更新资源引用。
- 发布策略：
  - 先发布带 hash 的静态资源
  - 再发布引用新 hash 的 HTML

白屏常见原因：
- HTML 命中旧缓存，但 JS 已更新/删除；或 HTML 新但 JS 还没生效。
- CDN 回源延迟、节点缓存不一致。

工程手段：
- 资源 404/加载失败兜底（重试/降级页）。
- 版本探测与强制刷新提示（谨慎使用）。
