# 题目名
代码分割与按需加载策略

## 题目描述
什么是代码分割（code splitting）？常见的按需加载策略有哪些（路由级、组件级、第三方库拆分）？如何权衡“首屏体积”与“请求数量/缓存复用”？

## 题目答案
代码分割：把一个大 bundle 拆成多个 chunk，在需要时再加载，从而降低首屏下载与解析成本。

常见策略：
- 路由级拆分：按页面/路由维度拆包，最常见。
- 组件级懒加载：大组件、低频组件在交互触发时加载。
- 第三方依赖拆分：把稳定的大依赖拆成单独 chunk，提高缓存命中（但要注意不要过度拆分）。

权衡点：
- chunk 太大：首屏慢、解析慢。
- chunk 太碎：请求数增多（HTTP/2/3 缓解但仍有开销）、瀑布加载可能变慢。
- 缓存：
  - 对稳定依赖做长缓存（内容哈希），提升复用
  - 对业务代码按路由拆分，可减少每次发布导致的缓存失效范围

工程实践：
- 以 RUM（真实用户监控）指标驱动：首屏 FCP/LCP、交互 INP、资源加载瀑布。
- 关键页面可做预加载/预取（prefetch/preload），减少用户等待。
