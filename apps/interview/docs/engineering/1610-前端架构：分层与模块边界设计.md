# 题目名
前端架构：分层与模块边界设计

## 题目描述
大型前端项目如何做分层与模块边界？常见分层（基础设施/领域/应用/展示）分别放什么？如何避免“循环依赖、巨型 util、跨层调用失控”？

## 题目答案
分层目标：降低耦合、提升可演进性，让依赖方向清晰可控。

常见分层（示例）：
- 基础设施层：请求封装、日志、配置、存储、通用组件库适配。
- 领域层：业务核心模型与规则（计算、校验、状态机），尽量纯逻辑。
- 应用层：用例编排（调用领域逻辑 + 基建 IO），路由/页面级 orchestration。
- 展示层：组件与视图渲染，尽量薄。

边界治理：
- 明确依赖方向：展示 → 应用 → 领域；基建可被上层依赖，但尽量不反向。
- 避免巨型 util：把工具按领域/模块归属，或下沉到明确的基础设施包。
- 防循环依赖：构建时检测、对外只暴露稳定入口（barrel 需谨慎），必要时引入中间层/事件机制解耦。

工程手段：
- 用 lint 规则/依赖图工具限制跨层导入。
- 统一模块入口与导出策略，减少深层路径引用。
