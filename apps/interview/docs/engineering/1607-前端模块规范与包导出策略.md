# 题目名
前端模块规范与包导出策略

## 题目描述
一个 npm 包应该如何设计导出（`main/module/types/exports`）？为什么要用 `exports` 字段？如何兼容 ESM/CJS 并避免深层路径随意导入？

## 题目答案
导出字段的目标：让不同运行时与工具链拿到正确的入口，并限制对内部实现的耦合。

常见字段：
- `main`：传统 CJS 入口。
- `module`：历史上给 bundler 的 ESM 入口（并非 Node 标准）。
- `types/typings`：类型声明入口。
- `exports`：现代推荐方式，可做条件导出与子路径导出控制。

为什么用 `exports`：
- 明确声明可被导入的路径，阻止消费者随意 `import 'pkg/lib/internal'` 绑死内部结构。
- 支持条件导出：按 `import/require/browser/node` 等选择不同入口。

兼容策略：
- 同时提供 ESM 与 CJS 构建产物，并在 `exports` 中按条件区分。
- 对外只暴露稳定 API，内部实现随版本演进。
