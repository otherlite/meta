# 题目名
前端测试体系：单元、组件与 E2E

## 题目描述
前端测试有哪些层级（单元测试、组件测试、E2E）？分别测什么、价值是什么？如何选择投入产出比最高的测试策略？

## 题目答案
测试层级：
- 单元测试：测试纯函数/工具函数/业务逻辑单元。
  - 价值：快、稳定、定位精确。
- 组件测试：测试组件渲染与交互（含状态、props、事件）。
  - 价值：覆盖 UI 行为与边界，成本中等。
- E2E：从用户视角跑完整流程（登录、下单等）。
  - 价值：覆盖集成与真实链路；缺点是慢、易受环境影响。

策略建议：
- “金字塔”：单元最多，组件其次，E2E 最少但覆盖关键路径。
- 优先对高风险、高变更、高价值模块加测试（权限、路由、支付、核心表单）。
- 对难测的模块，先做可测试性改造（逻辑与视图分离、依赖注入）。
