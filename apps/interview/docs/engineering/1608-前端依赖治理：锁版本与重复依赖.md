# 题目名
前端依赖治理：锁版本与重复依赖

## 题目描述
为什么需要 lockfile？重复依赖/依赖提升会带来什么问题？在 monorepo 中如何治理依赖版本漂移与幽灵依赖？

## 题目答案
lockfile 的价值：
- 固定依赖树，保证不同机器/不同时间安装结果一致，减少“我这能跑你那不能跑”。

重复依赖问题：
- 包体变大、构建变慢。
- 运行时多实例风险（例如 react 多份会导致 hooks 失效）。

治理手段：
- 统一版本策略：在 workspace 根约束关键依赖版本。
- 定期做依赖审计与去重（按工具链能力）。
- 避免幽灵依赖：保证每个包显式声明所用依赖。
- 对关键依赖加校验（例如构建时检查 react 版本单一）。
