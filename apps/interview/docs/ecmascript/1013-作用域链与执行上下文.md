# 题目名
作用域链与执行上下文

## 题目描述
请解释“作用域（scope）”“作用域链”“执行上下文（execution context）”分别是什么，它们的关系是什么？变量提升与 TDZ（暂时性死区）如何影响访问时机？

## 题目答案
作用域：变量/函数的可访问范围，由代码的词法结构决定（词法作用域）。

作用域链：
- 当访问一个标识符时，JS 引擎会先查找当前作用域的绑定，没有则沿外层作用域逐级查找直到全局。

执行上下文：
- 代码执行时的运行环境记录，包含：
  - 变量环境（var/函数声明）
  - 词法环境（let/const/class）
  - this 绑定
  - 外部环境引用（连接到外层上下文形成链）

关系：
- 作用域在“写代码时”确定；执行上下文在“运行时”创建。
- 作用域链的查找依赖执行上下文持有的外部环境引用。

变量提升与 TDZ：
- `var` 声明会提升并初始化为 `undefined`，因此在声明前访问不会报错但值是 `undefined`。
- `let/const` 会提升但不会初始化，在声明前访问处于 TDZ，会抛 `ReferenceError`。
