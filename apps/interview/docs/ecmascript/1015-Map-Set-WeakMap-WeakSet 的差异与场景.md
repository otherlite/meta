# 题目名
`Map/Set/WeakMap/WeakSet` 的差异与场景

## 题目描述
对比 `Map`、`Set`、`WeakMap`、`WeakSet`：键值类型限制、可枚举性、GC 语义分别是什么？工程中典型应用场景有哪些？

## 题目答案
`Map`：键值对集合，key 可以是任意类型（对象、函数、原始值）。可遍历、可枚举。

`Set`：值的集合，保证唯一。可遍历、可枚举。

`WeakMap`：
- key 只能是对象（或函数），value 任意。
- key 是弱引用：如果外界不再引用该 key 对象，GC 可回收该对象，同时对应条目也会消失。
- 不可枚举（无法遍历所有 key）。

`WeakSet`：
- 值只能是对象，且是弱引用。
- 不可枚举。

典型场景：
- `Map`：缓存计算结果（key 可为对象）、管理对象到元信息的映射。
- `Set`：去重、记录已处理元素、依赖收集集合。
- `WeakMap`：与对象生命周期绑定的缓存（避免内存泄漏），如把 DOM 节点映射到私有数据。
- `WeakSet`：记录“已标记/已访问”的对象且不阻止回收。
