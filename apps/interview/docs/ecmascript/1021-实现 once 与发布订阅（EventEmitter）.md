# 题目名
实现 `once` 与发布订阅（EventEmitter）

## 题目描述
实现一个 `once(fn)`：只允许函数执行一次，之后调用返回第一次的结果。再实现一个简化的 EventEmitter：支持 `on/off/emit/once`。

## 题目答案
`once` 核心：
- 用闭包保存 `called` 标志与 `result`。
- 首次调用执行并缓存结果；后续调用直接返回缓存结果。

EventEmitter 核心：
- 用 `Map<string, Set<Function>>` 或对象映射维护事件到监听器集合。
- `on`：注册监听器。
- `off`：移除监听器（需要能精确匹配同一函数引用）。
- `emit`：复制一份监听器列表再遍历执行，避免执行过程中增删导致迭代异常。
- `once`：注册一个包装函数，内部先 `off` 自己再执行原回调。

工程注意：
- 监听器异常不应阻止其他监听器执行（可按需求 try/catch）。
- 防止内存泄漏：长生命周期 emitter 应支持清理全部监听器。
