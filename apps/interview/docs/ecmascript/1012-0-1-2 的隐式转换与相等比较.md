# 题目名
`0.1 + 0.2 !== 0.3` 的原因与相等比较

## 题目描述
为什么 `0.1 + 0.2 !== 0.3`？如何在工程中安全地做浮点数比较与运算？隐式类型转换在 `==` 中的规则有哪些需要警惕？

## 题目答案
原因：JavaScript 使用 IEEE 754 双精度浮点数表示小数，很多十进制小数无法被二进制有限表示，只能存近似值。

- `0.1`、`0.2` 在二进制里都是无限循环小数，存储时被截断成近似值。
- 相加后误差累积，得到 `0.30000000000000004`，所以严格相等失败。

工程处理：
- 比较：用误差阈值（epsilon）
  - 判断 `Math.abs(a - b) < Number.EPSILON`（或按业务放大阈值）
- 金额/计数：用整数表示最小单位（分、毫秒），避免浮点。
- 需要高精度：用大数库/Decimal 库（按项目依赖约束选择）。

`==` 的坑（面试点）：
- `==` 会触发隐式类型转换，规则复杂，易出边界问题。
- 典型：
  - `'' == 0` 为 true
  - `false == 0` 为 true
  - `null == undefined` 为 true（但它们与其他值比较通常为 false）

结论：
- 业务代码优先使用 `===`，必要时显式转换后再比较。
