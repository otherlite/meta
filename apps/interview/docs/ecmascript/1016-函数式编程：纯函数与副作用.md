# 题目名
函数式编程：纯函数与副作用

## 题目描述
什么是纯函数？什么是副作用？为什么“纯函数 + 不可变数据”有利于可测试性与可维护性？在前端工程中有哪些实际落地方式？

## 题目答案
纯函数：
- 相同输入必定得到相同输出。
- 不依赖也不修改函数外部的可变状态。

副作用：
- 函数执行对外部世界产生影响，如修改全局变量、读写 DOM、发请求、写日志、读时间/随机数等。

优势：
- 可测试：不需要复杂的环境搭建与 mock，输入输出即可验证。
- 易组合：函数可以像积木一样组合（map/filter/reduce）。
- 易推理：减少“隐藏状态”带来的复杂度。

工程落地：
- 把副作用放到边界层（请求层、UI 层），业务计算尽量写成纯逻辑函数。
- 用不可变更新（如拷贝更新、结构共享）降低状态错改风险。
- 在状态管理中保持单向数据流，把 IO 放在 effect 层。
