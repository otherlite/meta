# 题目名
深拷贝与浅拷贝（循环引用与特殊类型）

## 题目描述
深拷贝与浅拷贝的区别是什么？如何实现一个可用的深拷贝，需要处理哪些边界（循环引用、Date/RegExp/Map/Set/函数等）？

## 题目答案
浅拷贝：只复制第一层属性的引用，嵌套对象仍共享同一引用。常见：`Object.assign`、展开运算符（对对象/数组）。

深拷贝：递归复制，目标对象与源对象在嵌套层级也不共享引用。

实现深拷贝时的关键点：
- 循环引用：必须用缓存表记录“源对象 -> 克隆对象”的映射，避免无限递归（通常用 `WeakMap`）。
- 特殊内建类型：
  - `Date`：`new Date(old.getTime())`
  - `RegExp`：复制 `source`、`flags`、`lastIndex`
  - `Map/Set`：需要遍历并深拷贝 key/value 或 value
  - `ArrayBuffer`/TypedArray：根据业务是否需要复制底层缓冲
- 函数：一般不做深拷贝（函数是可执行代码与闭包环境），常见策略是“原样返回引用”。
- 原型与属性描述符：更“完整”的深拷贝需要保留原型链与 `get/set`、`writable` 等描述符（工程中通常按需要取舍）。

工程建议：
- 若只是数据传输/持久化：优先使用结构化克隆（浏览器 `structuredClone`）或序列化（但 JSON 会丢类型、丢函数、丢 `undefined`，并会把 `NaN/Infinity` 变成 `null`）。
