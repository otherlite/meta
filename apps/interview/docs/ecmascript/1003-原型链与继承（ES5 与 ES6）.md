# 题目名
原型链与继承（ES5 与 ES6）

## 题目描述
请解释 JavaScript 的原型链是什么？`new` 做了哪些事？ES5 常见继承方式有哪些缺陷？ES6 `class extends` 底层本质是什么？

## 题目答案
原型链：每个对象都有内部指针指向其原型对象（`[[Prototype]]`，可通过 `Object.getPrototypeOf(obj)` 读取），当访问属性时，如果对象本身没有，会沿着原型链向上查找，直到 `null`。

`new Foo(...args)` 的步骤（概念上）：
1. 创建一个新对象 `obj`
2. 设置原型：`obj.[[Prototype]] = Foo.prototype`
3. 执行构造函数：`Foo.apply(obj, args)`
4. 若构造函数返回一个对象，则返回该对象；否则返回 `obj`

ES5 常见继承：
- 原型链继承：`Child.prototype = new Parent()`
  - 缺陷：父类引用类型属性被所有实例共享；无法向父类构造传参。
- 构造函数继承：`Parent.call(this, ...)`
  - 优点：可传参，避免共享引用属性；缺陷：方法无法复用（每个实例一份）。
- 组合继承：构造函数继承 + 原型链继承
  - 缺陷：父类构造函数会执行两次（一次给子类原型，一次给实例）。
- 寄生组合继承：`Child.prototype = Object.create(Parent.prototype)` 再修正 `constructor`
  - 这是 ES5 时代最推荐的继承方式。

ES6 `class extends` 本质仍是原型链：
- 子类原型对象的 `[[Prototype]]` 指向父类原型：`Child.prototype.__proto__ === Parent.prototype`
- 子类构造函数自身的 `[[Prototype]]` 指向父类构造函数：`Child.__proto__ === Parent`
- `super()` 等价于在子类构造中调用父类构造，并把 `this` 初始化好（在子类构造中使用 `this` 前必须先 `super()`）。
