# 题目名
Proxy 与 Reflect 的典型应用

## 题目描述
`Proxy` 能解决哪些问题？`Reflect` 的作用是什么？请给出 2~3 个工程中的典型使用场景，并说明注意事项。

## 题目答案
`Proxy`：为对象提供一层“代理”，可拦截 `get/set/has/deleteProperty/ownKeys` 等操作。

`Reflect`：提供与对象内部方法对应的一组静态函数（如 `Reflect.get/set/has`），常用于在 Proxy 中以更规范的方式执行默认行为，并返回布尔值/结果，避免手写 `try/catch` 或隐式异常。

典型场景：
- 响应式系统：拦截读取做依赖收集（track），拦截写入做派发更新（trigger），Vue3 即是典型例子。
- 数据校验/防御式编程：
  - `set` 时校验类型、范围，非法写入直接返回 `false` 或抛错。
  - 读取不存在属性给出更友好的错误提示。
- 观察/埋点：
  - 统计某些属性访问频次（注意不要引入显著性能开销）。

注意事项：
- 代理对象与原对象不全等，某些场景需要缓存代理避免重复创建。
- 过度拦截会引入性能成本；热路径谨慎使用。
- 对象的可枚举性/原型链行为要谨慎处理（如 `ownKeys` 影响 `Object.keys`）。
