# 题目名
防抖与节流的区别与实现

## 题目描述
防抖（debounce）与节流（throttle）分别解决什么问题？有什么差异？请给出实现要点与适用场景。

## 题目答案
二者都用于控制高频事件触发带来的性能问题（scroll、resize、mousemove、input）。

防抖（debounce）：
- 目标：在“连续触发”结束后只执行一次。
- 行为：每次触发都重置计时器，直到停止触发超过 `wait` 才执行。
- 适用：搜索框联想、窗口 resize 后重新计算布局、用户停止输入后提交校验。
- 关键点：是否需要“立即执行（leading）”与“结束再执行（trailing）”。

节流（throttle）：
- 目标：在持续触发期间，按固定时间间隔最多执行一次。
- 行为：每 `wait` 时间窗口执行一次，窗口内的触发被忽略或合并。
- 适用：滚动监听上报、拖拽过程计算、mousemove 过程更新 UI。
- 关键点：用时间戳或定时器实现；同样考虑 leading/trailing。

工程实践：
- 对需要“最终状态一定执行”的场景（如滚动停止后对齐吸附），要开启 trailing。
- 对输入类交互往往更偏向 debounce。
