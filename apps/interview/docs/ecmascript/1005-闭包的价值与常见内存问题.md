# 题目名
闭包的价值与常见内存问题

## 题目描述
什么是闭包？闭包有什么工程价值？什么时候会引发内存占用问题？如何定位与治理？

## 题目答案
闭包：函数与其“定义时的词法环境”的组合。即使外层函数已经返回，内层函数仍能访问外层作用域中的变量。

工程价值：
- 封装私有状态（模块化、信息隐藏）
- 函数工厂/偏函数：把配置“固化”成可复用函数
- 事件处理/异步回调中保留上下文

常见内存问题并非“闭包一定泄漏”，而是“闭包持有的引用导致对象长期不可达释放”：
- 闭包引用了体积很大的对象（DOM 节点、缓存、数据列表），且闭包被长生命周期对象引用（全局变量、单例、订阅回调）
- 事件监听/订阅未解除导致回调与其闭包无法释放

定位思路：
- 使用 Chrome DevTools Memory：
  - Heap Snapshot：查找 Retainers（是谁引用住了对象）
  - Allocation instrumentation on timeline：观察持续增长
- 常见信号：页面多次进入/离开后内存持续上涨；节点数不回落；Detached DOM tree

治理手段：
- 解绑事件/取消订阅/清理定时器（`removeEventListener`、`clearInterval`）
- 缩小闭包捕获范围：不要在闭包里引用不必要的大对象
- 使用弱引用结构（如 `WeakMap`）存放与对象生命周期绑定的缓存
- 控制缓存上限（LRU/TTL）
