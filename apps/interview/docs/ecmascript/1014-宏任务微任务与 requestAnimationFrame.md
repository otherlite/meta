# 题目名
宏任务/微任务与 `requestAnimationFrame`

## 题目描述
`requestAnimationFrame`（rAF）处在事件循环的什么位置？它和 `setTimeout`、微任务（Promise）之间的执行顺序如何？为什么动画推荐用 rAF？

## 题目答案
在浏览器里一次循环通常是：执行宏任务 → 清空微任务 → 可能渲染（含 rAF 回调）→ 下一轮宏任务。

`requestAnimationFrame`：
- rAF 回调会在“下一次绘制前”执行，通常发生在微任务清空之后、绘制之前。
- 因此 rAF 非常适合做动画：能与刷新率对齐，避免无意义的多次绘制。

与 `setTimeout` 的差异：
- `setTimeout(fn, 0)` 会把 `fn` 放入宏任务队列，不保证与渲染同步，且最小延迟受浏览器限制。
- rAF 由浏览器在渲染前调度，能更平滑。

与微任务的关系：
- 微任务会在当前宏任务结束后立即清空；如果微任务过多，会推迟 rAF 与渲染，导致掉帧。
