# 题目名
CSRF 原理与 SameSite 策略

## 题目描述
什么是 CSRF？为什么“用户没打开站点也可能被攻击”？常见防护手段有哪些？SameSite Cookie 能解决多少问题？

## 题目答案
CSRF（Cross-Site Request Forgery）：利用浏览器会自动携带目标站点 Cookie 的特性，诱导用户在已登录状态下向目标站点发起非本意请求（如转账、改密码）。

为什么用户“没打开站点也可能被攻击”：
- 用户只要在目标站点保持登录态（Cookie 仍有效），访问攻击站点时，攻击站点可以通过表单提交、图片请求等方式让浏览器向目标站点发请求，浏览器会自动带上目标站点的 Cookie。

防护手段：
- CSRF Token：
  - 服务端下发随机 token（与用户会话绑定），写入表单/请求头
  - 服务端校验 token，攻击站点无法读到 token（同源策略）
- SameSite Cookie：
  - `SameSite=Lax/Strict/None` 控制跨站请求是否携带 Cookie
  - `None` 必须配合 `Secure`
  - 能显著降低风险，但对一些跨站业务（如第三方登录回跳）需要额外设计
- 验证 Referer/Origin：
  - 服务端校验请求来源（注意兼容与隐私限制，不能作为唯一手段）
- 关键操作二次校验：验证码/短信/密码确认等（增加攻击成本）

工程结论：
- Token + SameSite 是常见组合；对敏感操作可叠加二次校验。
