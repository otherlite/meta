# 题目名
开放重定向（Open Redirect）与钓鱼风险

## 题目描述
什么是开放重定向？为什么它常被用于钓鱼？前端与后端应如何防护（白名单、签名、相对路径限制等）？

## 题目答案
开放重定向：站点提供跳转能力（如 `?redirect=`），但未校验目标地址，导致攻击者可把用户从可信域名跳到恶意站点。

风险：
- 用户看到的是可信域名链接，点击后被重定向到钓鱼页。
- 常与 OAuth 回跳、登录后跳转结合利用。

防护：
- 目标地址白名单（只允许站内路径或特定域名）。
- 只允许相对路径（禁止 `http(s)://` 绝对外链）。
- 对跳转参数做签名（防篡改），服务端校验。
- 对外链跳转增加提示页（降低误点）。
