# 题目名
SRI（Subresource Integrity）子资源完整性

## 题目描述
SRI 是什么？它解决什么问题？如何给 `<script>`/`<link>` 配置 SRI？它与 CSP 的关系是什么，有哪些限制？

## 题目答案
SRI 用于校验静态资源内容是否被篡改：浏览器会对下载的资源做哈希校验，只有匹配 `integrity` 声明才执行/应用。

使用方式：
- 在 `<script>`/`<link>` 上添加 `integrity="sha256-..."`（或 sha384/sha512）。
- 跨域资源通常需要配合 `crossorigin`（如 `anonymous`）。

解决的问题：
- CDN/第三方资源被篡改时，避免加载恶意脚本。

与 CSP：
- CSP 控制“允许从哪里加载”，SRI 控制“加载的内容是否是预期内容”。
- 二者可组合使用以增强安全性。

限制：
- 资源内容变更必须同步更新 hash，动态内容不适合。
