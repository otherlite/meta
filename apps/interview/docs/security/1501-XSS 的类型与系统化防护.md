# 题目名
XSS 的类型与系统化防护

## 题目描述
什么是 XSS？反射型、存储型、DOM 型 XSS 有什么区别？工程上如何系统化防护（输入/输出、CSP 等）？

## 题目答案
XSS（Cross-Site Scripting）：攻击者把恶意脚本注入页面，使其在受害者浏览器中执行，从而窃取 Cookie、劫持会话、篡改页面等。

类型：
- 反射型：恶意 payload 来自 URL/请求参数，服务端把它“反射”到响应 HTML 中。
- 存储型：payload 被保存到数据库/评论区等，后续访问页面时触发，影响面更大。
- DOM 型：不经过服务端输出拼接，前端脚本在 DOM 操作中把不可信数据写进可执行上下文导致。

防护思路（系统化）：
- 核心原则：永远把外部输入当成不可信数据。
- 输出编码（首选）：根据上下文做正确转义：
  - HTML 文本节点、HTML 属性、URL、JS 字符串、CSS 等上下文的编码策略不同
- 避免危险 API：`innerHTML`、`document.write`、`insertAdjacentHTML` 等；尽量使用 `textContent`、DOM API 创建节点。
- Cookie 防护：设置 `HttpOnly`（阻止 JS 读取）、`Secure`、`SameSite`。
- CSP（Content-Security-Policy）：限制脚本来源、禁用 inline script（配合 nonce/hash），降低注入成功后的执行概率。
- 富文本白名单：如果必须渲染富文本，使用可信的 sanitize（白名单）方案，不要黑名单。
