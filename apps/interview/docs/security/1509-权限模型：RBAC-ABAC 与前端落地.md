# 题目名
权限模型：RBAC / ABAC 与前端落地

## 题目描述
RBAC 与 ABAC 分别是什么？它们适用于什么场景？前端在权限体系里应该做哪些事、不应该做哪些事？如何设计路由/按钮级权限控制？

## 题目答案
RBAC（Role-Based Access Control）：
- 以角色为中心：用户 → 角色 → 权限。
- 优点：易理解、易维护；缺点：细粒度场景容易角色爆炸。

ABAC（Attribute-Based Access Control）：
- 以属性与策略为中心：基于用户属性、资源属性、环境属性（时间、地域）计算是否允许。
- 优点：细粒度、表达力强；缺点：策略复杂度与治理成本更高。

前端应该做：
- 体验层权限：隐藏/禁用无权限按钮、路由守卫、菜单裁剪。
- 统一的权限判定函数（例如 `can(action, resource)`），避免散落 if。

前端不应该做：
- 把前端当作最终权限边界。真正的鉴权必须由服务端强制执行。

路由/按钮落地：
- 路由：进入前校验权限，不通过则跳转 403/登录。
- 按钮：用权限点渲染或禁用，关键操作仍依赖服务端返回的 403 做兜底。
