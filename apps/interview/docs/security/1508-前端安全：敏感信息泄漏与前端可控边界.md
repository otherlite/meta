# 题目名
敏感信息泄漏与前端可控边界

## 题目描述
前端工程中常见的敏感信息泄漏渠道有哪些（源码、SourceMap、日志、配置、接口返回）？哪些安全问题前端无法单独解决？如何建立“前端可控边界”？

## 题目答案
常见泄漏渠道：
- 把密钥/内网地址/管理接口写进前端代码或构建产物。
- SourceMap 公开托管导致源码与注释暴露。
- 前端日志/埋点把用户隐私或 token 上报。
- 接口返回过多字段（越权信息）被前端展示或被抓包获取。

前端无法单独解决的点：
- 鉴权与权限控制必须由服务端强制执行（前端校验只能做体验层）。
- 数据脱敏与最小化返回由后端决定。

建立可控边界：
- 前端只存放公开信息与短期状态，不存长期敏感凭证。
- 生产环境管控 SourceMap、关闭调试信息暴露。
- 上报与日志做字段白名单与脱敏。
- 与后端约定权限模型与错误码语义，避免前端“猜测权限”。
