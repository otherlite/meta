# 题目名
前端接口设计：分页、幂等与重试

## 题目描述
从前端视角看，接口设计里哪些点最影响可用性与体验？分页（offset/cursor）、幂等、重试与超时如何协同？

## 题目答案
分页：
- offset 分页：实现简单，但数据变化时可能出现跳页/重复。
- cursor 分页：基于游标（时间/ID），更稳定，适合流式列表与大数据。

幂等：
- 对写操作建议支持幂等键（Idempotency-Key）或业务去重，避免重试造成重复创建/扣款。

重试与超时：
- 网络抖动可重试，但要区分幂等与非幂等请求。
- 超时需要结合取消（如 AbortController），避免无效请求堆积。
- 重试要做退避（指数退避 + 抖动），避免雪崩。

工程侧：
- 明确错误码语义与可重试策略，前端才能做一致的兜底体验。
