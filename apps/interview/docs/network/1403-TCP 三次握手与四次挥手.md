# 题目名
TCP 三次握手与四次挥手

## 题目描述
请说明 TCP 三次握手与四次挥手的过程与目的。为什么握手是三次、挥手通常是四次？TIME_WAIT 的意义是什么？

## 题目答案
三次握手（建立连接）：
- 目标：确认双方收发能力正常，并同步初始序列号。
1. 客户端 → 服务端：SYN（我想建立连接，seq=x）
2. 服务端 → 客户端：SYN+ACK（我同意，seq=y，ack=x+1）
3. 客户端 → 服务端：ACK（收到，ack=y+1）

为什么三次：
- 需要双方都确认“自己能发、对方能收”；两次不足以让客户端确认服务端收发都正常并建立可靠状态。

四次挥手（关闭连接）：
- TCP 是全双工，关闭需要双方分别关闭各自的发送方向。
- 一方发送 FIN 表示“我不再发了”，对方 ACK 确认；对方也可能稍后再发 FIN。

TIME_WAIT：
- 主动关闭方进入 TIME_WAIT，等待 2MSL。
- 意义：
  - 确保最后的 ACK 能被对方收到（若丢失可重传）。
  - 让旧连接的延迟报文在网络中自然消失，避免影响后续同四元组的新连接。
