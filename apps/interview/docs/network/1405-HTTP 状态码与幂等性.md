# 题目名
HTTP 状态码与幂等性

## 题目描述
请总结常见 HTTP 状态码（200/301/302/304/400/401/403/404/409/429/500/502/503）含义，并解释什么是幂等性。哪些方法幂等？幂等对重试有什么意义？

## 题目答案
常见状态码：
- 200：成功。
- 301/302：重定向（301 永久，302 临时，实践中还要关注 307/308 的语义）。
- 304：协商缓存命中，资源未修改。
- 400：客户端请求错误（参数不合法等）。
- 401：未认证（通常需要登录/携带凭证）。
- 403：已认证但无权限。
- 404：资源不存在。
- 409：冲突（资源版本冲突、重复提交等）。
- 429：请求过多（限流）。
- 500：服务端内部错误。
- 502：网关错误（上游服务异常）。
- 503：服务不可用（过载/维护）。

幂等性：
- 同一个请求执行一次或多次，对资源造成的“最终效果”一致。

常见方法：
- 幂等：GET/HEAD/PUT/DELETE（语义上幂等；实现上要看服务端是否严格遵守）。
- 非幂等：POST（通常用于创建/触发动作，多次提交可能产生多份结果）。

对重试意义：
- 网络超时等不确定失败时，幂等请求更适合自动重试。
- 对非幂等请求，重试要配合幂等键（Idempotency-Key）或业务去重，避免重复创建/扣款。
