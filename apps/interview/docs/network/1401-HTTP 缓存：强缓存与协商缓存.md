# 题目名
HTTP 缓存：强缓存与协商缓存

## 题目描述
请说明浏览器的 HTTP 缓存策略：强缓存与协商缓存分别是什么？相关的请求/响应头有哪些？刷新行为（普通刷新、强制刷新）分别会怎样？

## 题目答案
HTTP 缓存目标：减少网络请求、降低延迟与带宽。

强缓存（不发请求或不带条件请求直接用本地）：
- 响应头：
  - `Cache-Control: max-age=...`（HTTP/1.1 推荐）
  - `Expires`（HTTP/1.0，受客户端时间影响）
- 命中强缓存时，浏览器直接使用本地副本。

协商缓存（发请求但用条件判断是否可复用）：
- 响应头/校验器：
  - `ETag`（实体标签，推荐）
  - `Last-Modified`（资源最后修改时间）
- 请求头：
  - `If-None-Match: <etag>` 对应 ETag
  - `If-Modified-Since: <time>` 对应 Last-Modified
- 若资源未变更，服务器返回 `304 Not Modified`，响应体通常为空，浏览器复用缓存。

优先级与组合：
- `Cache-Control` 优先级通常高于 `Expires`。
- 实践上可同时用 `Cache-Control` + `ETag`：
  - 在 `max-age` 有效期内走强缓存
  - 过期后走协商缓存

刷新行为（不同浏览器细节略有差异，面试答主线即可）：
- 普通刷新：可能跳过强缓存，带条件请求去协商（尽量拿到最新）。
- 强制刷新（Ctrl/⌘ + Shift + R）：倾向于跳过缓存，直接重新请求。
