# 题目名
DNS 解析与 DNS 污染/劫持

## 题目描述
DNS 解析的基本流程是什么？为什么会有 DNS 缓存？DNS 污染/劫持是什么，可能带来什么风险？前端与工程侧有哪些缓解手段？

## 题目答案
DNS 流程（概念）：
- 浏览器缓存 → 系统缓存 → 路由器/运营商缓存 → 递归解析器 → 权威 DNS。

缓存的意义：
- 降低延迟与解析压力，通过 TTL 控制缓存时长。

DNS 污染/劫持：
- 解析结果被篡改，把域名指向错误/恶意 IP。
- 风险：钓鱼、注入广告脚本、中间人攻击（若未正确校验 TLS）。

缓解手段：
- 全站 HTTPS + 严格证书校验（降低被劫持后内容被篡改的影响）。
- 使用可信 DNS/DoH/DoT（在可控环境下）。
- CDN 与多线路容灾，异常时快速切换。
- 对关键资源启用 SRI（子资源完整性）与 CSP，降低被注入风险。
