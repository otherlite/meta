# 题目名
CDN 的工作原理与缓存策略

## 题目描述
CDN 为什么能加速？回源是什么？CDN 缓存命中/回源如何影响性能？前端静态资源发布如何配合 CDN 做缓存治理？

## 题目答案
CDN 加速原理：
- 把内容缓存到离用户更近的边缘节点，减少 RTT 与跨网传输。

回源：
- 当边缘节点没有缓存或缓存过期，会向源站请求资源并缓存。

缓存策略关键：
- 静态资源用内容哈希命名 + 长缓存（immutable），命中率高且可控失效。
- HTML 入口短缓存或 `no-cache`，让入口及时更新到新 hash。
- 对接口响应可按业务决定是否可缓存（并谨慎处理鉴权与个性化）。

性能影响：
- 命中边缘缓存：响应快。
- 回源：延迟增加且源站压力上升。
