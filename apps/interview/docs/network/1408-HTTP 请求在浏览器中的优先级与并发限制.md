# 题目名
HTTP 请求在浏览器中的优先级与并发限制

## 题目描述
浏览器对资源请求有并发限制吗？不同资源类型（CSS/JS/图片/字体）是否有加载优先级差异？HTTP/2/3 多路复用后并发限制还重要吗？

## 题目答案
并发限制：
- HTTP/1.1 时代浏览器对同域并发连接数有限制（不同浏览器略有差异），过多资源会排队。

优先级差异：
- CSS/关键 JS 可能影响渲染与执行，通常优先级更高。
- 字体与首屏大图会影响 LCP/CLS，优先级与预加载策略相关。

HTTP/2/3：
- 多路复用让一个连接可并发多个请求，减少“连接数限制”带来的阻塞。
- 但仍存在：带宽竞争、服务器优先级实现差异、主线程解析/执行瓶颈。

工程实践：
- 用 `preload` 提升关键资源优先级。
- 避免首屏同时拉取过多非关键资源。
