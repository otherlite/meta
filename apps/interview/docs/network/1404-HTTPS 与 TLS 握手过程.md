# 题目名
HTTPS 与 TLS 握手过程

## 题目描述
HTTPS 为什么安全？TLS 握手大致做了什么（证书校验、密钥协商）？对称加密与非对称加密各自负责什么？

## 题目答案
HTTPS = HTTP + TLS。TLS 主要提供：
- 机密性：内容加密传输。
- 完整性：防篡改（MAC/AEAD）。
- 身份认证：证书体系确保你连的是“目标站点”。

对称/非对称分工：
- 非对称加密：用于握手阶段的身份认证与密钥协商（成本高，不适合大量数据）。
- 对称加密：用于握手完成后的数据传输（性能高）。

握手阶段（概念版）：
- ClientHello：客户端提供支持的版本、加密套件、随机数等。
- ServerHello：服务端选择参数并返回证书链。
- 客户端校验证书：
  - 验证链路到可信根 CA
  - 验证域名匹配、有效期、吊销状态等
- 密钥协商：
  - 现代 TLS 常用 ECDHE 等（具备前向安全性），双方协商出会话密钥。
- Finished：双方确认握手消息未被篡改，进入加密通信。

面试加分点：
- 前向安全性（PFS）：即使服务器私钥泄漏，历史会话也难以被解密（取决于密钥协商算法）。
