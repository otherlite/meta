# 题目名
HTTP/2 多路复用为何仍会被主线程阻塞

## 题目描述
HTTP/2 已经多路复用了，为什么页面加载仍可能很慢？“网络并发”与“主线程瓶颈”如何相互影响？

## 题目答案
HTTP/2 解决的是“连接层面的并发”，但页面性能还受多种因素影响。

常见原因：
- 主线程阻塞：JS 解析/执行、长任务、频繁布局绘制会推迟渲染与交互。
- 资源优先级：关键 CSS/JS 未被优先加载或未被正确 preload。
- 依赖瀑布：即使网络并发，代码分割不当或运行时动态导入导致瀑布加载。
- 服务器端瓶颈：TTFB 高、回源慢、CDN 未命中。

工程结论：
- 网络层优化（HTTP/2/3）与主线程优化（减少 long task、拆分代码、优化渲染）需要一起做。
