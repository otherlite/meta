# 题目名
HTTP/1.1、HTTP/2、HTTP/3 的核心差异

## 题目描述
HTTP/1.1 相比 HTTP/2 的提升点有哪些？HTTP/2 还存在哪些问题？HTTP/3 为什么改用 QUIC？

## 题目答案
HTTP/1.1 的主要问题：
- 多个请求复用一个 TCP 连接时存在队头阻塞（应用层层面：同一连接内响应必须按顺序返回）。
- 浏览器通常开多个并发连接提升吞吐，但连接数受限且成本高。

HTTP/2 的核心改进：
- 二进制分帧：把数据切成帧，提升传输效率。
- 多路复用：同一连接上并发多个 stream，请求/响应交错传输。
- 头部压缩（HPACK）：减少重复 header 体积。
- 服务器推送（Server Push，实际使用中有争议，很多场景已不推荐）。

HTTP/2 的痛点：
- TCP 层的队头阻塞仍然存在：丢包会导致整个连接的所有 stream 受影响。

HTTP/3（基于 QUIC/UDP）：
- QUIC 在 UDP 之上实现可靠传输与拥塞控制，且把“多路复用”下沉到传输层，每个 stream 独立处理丢包，减少 TCP 队头阻塞影响。
- 连接建立更快：把 TLS（通常是 TLS 1.3）握手与传输层结合，支持 0-RTT/1-RTT。
- 更适合移动网络（切网、弱网）场景。
